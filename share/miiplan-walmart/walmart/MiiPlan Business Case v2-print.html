The Problem

Every day, 4,600 stores run suboptimal pick routes — and the cost compounds at scale.

OGP pick sequencing today handles one order at a time. No system optimizes across multiple orders simultaneously. Associates walk more than necessary every shift. At $72K in recoverable savings per store annually, the fleet-wide gap is $331M+.

Walmart has invested in WMS infrastructure, but OGP teams still can't minimize picker travel because single-order routing ignores the combinatorial problem. The gap grows with every increase in OGP volume.

Recommended Approach

An API overlay that makes the existing WMS smarter — proven across 217K line items.

  • 100-order batch optimization — multi-point constrained routing, not single-order heuristics
  • 2-day facility setup — proprietary digital twin from a CSV, not weeks of CAD
  • Replay/simulation — proves ROI against Walmart's own historical data before any commitment
  • Zero WMS disruption — API layer with per-store on/off toggle
  • Rigorous pilot design — 20 test / 20 control stores, randomized
  • Walmart-native expertise — Rob Jensen holds a Walton College PhD and spent 13 years at Walmart

Validated in a 3-month, 217,371 line-item case study — meets and exceeds requirements for an OGP pick-path optimization pilot.
